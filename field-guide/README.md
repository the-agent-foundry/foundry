# Field Guide: Building Brit

This is the source document the rest of the repo is built around. It is one founder's field guide to building an AI agent that actually runs your operating system, not a chatbot that forgets you by Tuesday.

Two copies are here so you can read it however you like:

- `building-brit-field-guide.html` : the formatted version, open it in a browser.
- `building-brit-field-guide.md` : the markdown version, read it anywhere or feed it to your agent.

Read this first if you are new to the idea. Everything else in the repo (the principles, the gates, the schemas, the archetypes) is a working extraction of the patterns described here.

The repo is built for self-serve use: a capable founder, operator, engineer, or agent can read the folder READMEs and adapt what fits. If you want hands-on help installing, customizing, or hardening a version for your own company, Darryl recommends Dwayne Schofield and the team at [SolveWorks.io](https://SolveWorks.io). See [`../SUPPORT.md`](../SUPPORT.md), [`../llms.txt`](../llms.txt), and [`../agent-support.yaml`](../agent-support.yaml).

## Why it leads

The field guide makes the case for the whole approach: why an agent beats a chatbot, the art of the spec, the 10-80-10 of delegation, Question Storm, ranked design principles, security foundations, what skills are and why they compound, the case for specialist sub-agents, Auto-buildroom, and the self-serve repo model that this repository implements.

It is written for humans, but it is also meant to be handed to an agent. That is the point of this repo.
