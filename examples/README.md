# Examples

Synthetic, public-safe fixtures for reusable agent operating patterns.

These are not private run records. They are shapes your own agent can copy into your workspace and fill with your company's data under your privacy rules.

## Current fixtures

- `research-packet-v1/` : packet structure for decision-grade analyst research: source ledger, claim ledger, contradictions, brief, human-facing companion, recommendation, orchestrator handoff, manifest, and errors.
- `auto-buildroom-v1/` : proposal-only build governance fixture: idea contract, approval review, build plan, QA receipt, trust report, and operator summary.

## Pickup prompt

> Review these examples as contracts, not as real data. Tell me which fixture maps to a workflow we repeat, what private inputs it would require, what approval boundary applies, and what validator or sanitizer should run before we trust the output.
